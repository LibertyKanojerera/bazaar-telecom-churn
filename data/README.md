# Data Directory

## ⚠️ Data Availability Notice

The raw dataset (`telecom.csv`) is **proprietary** and belongs to a Bulgarian telecommunications operator. It is **not included** in this repository.

To reproduce the analysis, you will need access to a dataset with the following schema.

---

## Schema

| Column | Type | Description |
|--------|------|-------------|
| `pid` | character | Business customer ID (dropped in preprocessing) |
| `crm_pid_value_segment` | categorical | Customer value tier: Bronze, Silver, Gold, Platinum, Iron, Lead, Other |
| `effective_segment` | categorical | Business size: SOHO, VSE, SME, SE, LE, Other |
| `billing_zip` | character | Postal/ZIP code of billing address (dropped) |
| `ka_name` | character | Key account manager initials (dropped) |
| `active_subscribers` | integer | Number of active SIM cards / subscriptions |
| `not_active_subscribers` | integer | Number of inactive subscribers (49.1% missing → imputed to 0) |
| `suspended_subscribers` | integer | Subscribers suspended in last 6 months (95.8% missing → imputed to 0) |
| `total_subs` | integer | Total subscriber count (dropped — multicollinear) |
| `avg_mobile_revenue` | numeric | Average monthly mobile revenue per customer |
| `avg_fix_revenue` | numeric | Average monthly fixed-line/internet revenue |
| `total_revenue` | numeric | Total revenue (dropped — multicollinear) |
| `arpu` | numeric | Average revenue per user (dropped — multicollinear) |
| `churn` | binary | Target variable: `Yes` = churned, `No` = retained |

---

## Processed Files (generated by `01_data_preparation.R`)

| File | Description |
|------|-------------|
| `telecom_encoded.rds` | Cleaned, dummy-encoded dataset for GLM/kNN/Tree |
| `telecom_rf.rds` | Cleaned dataset with factor variables for Random Forest |
| `training_idx.rds` | Row indices for the 60% training split (seed=12345) |
